# Attached Main Menu And Framework Sidebar

The app uses a persistent **Main menu** as the place where people start, and an attached **Framework sidebar** for contextual controls when the Framework tool is active. We chose an attached two-panel rail rather than putting all controls into the main menu or showing a detached settings panel, because resizing/collapsing navigation should not hide the active tool controls and the relationship between the selected tool and its controls should remain visible.
